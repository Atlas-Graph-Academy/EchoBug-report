- [ ] 确认目标组织有 Vercel Pro 账户访问权限
- [ ] 运行 `./backup-env.sh` 备份所有环境变量
- [ ] 记录当前 Vercel 项目名称：`_________________`
- [ ] 记录当前域名（如果有）：`_________________`
- [ ] 确认团队成员列表和所需权限

## 🔄 执行迁移

### GitHub 仓库迁移

- [ ] 在目标组织下创建新仓库（名称：`_________________`）
- [ ] 运行 `./migrate.sh [组织名]` 更新 Git 远程地址
- [ ] 验证代码已成功推送到新仓库
- [ ] 确认所有分支和标签已推送

### Vercel 部署迁移

- [ ] 登录到组织的 Vercel Pro 账户
- [ ] 创建新项目并导入新仓库
- [ ] 配置项目设置：
  - [ ] Framework: Next.js
  - [ ] Build Command: `npm run build`
  - [ ] Output Directory: `.next`
- [ ] 添加所有环境变量（从备份中）：
  - [ ] Production 环境变量
  - [ ] Preview 环境变量
  - [ ] Development 环境变量
- [ ] 配置自定义域名（如果有）
- [ ] 触发首次部署并验证成功

## ✅ 迁移后验证

### 功能测试

- [ ] 生产环境可以正常访问
- [ ] 所有 API 路由正常工作
- [ ] 环境变量正确加载
- [ ] 数据库连接正常（如果有）
- [ ] 第三方服务集成正常（Linear API 等）
- [ ] 认证功能正常

### 部署流程测试

- [ ] Push 到 main 分支触发自动部署
- [ ] 预览部署正常工作
- [ ] 构建日志无错误

### 权限和访问

- [ ] 团队成员可以访问新 GitHub 仓库
- [ ] 团队成员可以访问新 Vercel 项目
- [ ] 所有必要的权限已正确配置

## 🧹 清理（仅在确认一切正常后）

- [ ] 新部署已稳定运行至少 24 小时
- [ ] 所有功能测试通过

- [ ] （可选）删除个人账户下的旧 GitHub 仓库
- [ ] 删除本地备份文件（`vercel-env-backup/`）
